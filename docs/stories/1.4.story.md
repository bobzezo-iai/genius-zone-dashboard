## Story

**Como** cliente premium que já viu seu dashboard,
**Quero** fazer login com magic link (email) para acessar a Aurora continuamente,
**Para que** eu tenha uma mentora pessoal 24/7 com quota dedicada.

---

## Acceptance Criteria

- [ ] **AC1:** Botão discreto "Entrar para acessar Aurora" aparece no dashboard pré-renderizado
- [ ] **AC2:** Ao clicar, modal solicita email do cliente
- [ ] **AC3:** Supabase envia magic link por email (`supabase.auth.signInWithOtp({ email })`)
- [ ] **AC4:** Após login, Aurora usa quota de 20 msg/dia vinculada ao user_id do Supabase
- [ ] **AC5:** Sem login, Aurora funciona em modo anônimo com quota por session (funcionalidade existente)
- [ ] **AC6:** Login persiste entre sessões (Supabase session management)
- [ ] **AC7:** Ao fazer login, dados do localStorage migram de `client_{slug}` para `{supabase_user_id}`

---

## Dev Notes

- Supabase precisa estar configurado com magic link habilitado (Auth → Settings → Email)
- Se Supabase não configurado, botão de login não aparece (graceful degradation)
- Considerar como feature futura: vincular slug ao email do cliente no Supabase para validação
- Esta é SHOULD priority — o sistema funciona sem login (modo anônimo com Aurora)
